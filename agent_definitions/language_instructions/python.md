# Python language conventions for the implement agent

## Manifest & lockfile workflow

- `pyproject.toml` is committed to version control.
- `uv.lock` is **committed** to version control and is the source of
  truth for reproducible installs.
- **Never** hand-edit `uv.lock` — it is generated by `uv lock` from
  `pyproject.toml`.
- When `pyproject.toml` dependency lines change, the implement agent
  should note in its summary that a human must run `uv lock` and commit
  the updated lockfile. The sandbox runs with `--network none` and
  cannot run `uv lock`.
- CI (and local `make install`, and the Docker build) use
  `uv sync --frozen` which will fail if `uv.lock` is stale relative to
  `pyproject.toml` — this is intentional.

## Sandbox constraints (critical)

The implement sandbox runs with `--network none`. The agent **cannot**
run `uv sync`, `pip install`, `cargo build`, `uv lock`, or any command
that fetches from the network inside the sandbox. When package-manager
commands would fail due to lack of network:

- Commit the manifest change (e.g. `pyproject.toml` updated with a new
  dependency).
- In your summary, note that a human must run `uv lock` and commit the
  updated `uv.lock` (CI's `uv sync --frozen` will fail until they do).
- Do **not** `ask_user` or file a ticket for the inability to fetch
  packages — the operator expects the agent to note the required human
  step instead.
